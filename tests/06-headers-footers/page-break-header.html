This document demonstrates how to use pagebreak and pageheader elements. By default, both are invisible in the browser so that the page appears as a very long continuous page. But when printed, the continuous page is broken at the different location of pagebreak elements and the pageheader element is shown as well.

The class visible can be applied to those two elements to display them within the browser

On the generated PDF, the position of every "This is page #..." frame must be identical on all pages. The "pagebreak" and the "pageheader" SHALL NOT disturb the position of elements.

  • Between page #1 and page #2, the pagebreak and pageheader are not visible in the browser but are in the print media (PDF),
  • Between page #2 and page #3, the pagebreak and pageheader are visible in the browser and in the print media (PDF), thanks to the use of class visible

This is page #1.

A pagebreak and a pageheader have been inserted below. They are not visible in the browser. Thanks to the HTML code: <div class="pagebreak"></div> <div class="pageheader"> This is a "pageheader" </div>
(You cannot see either the pagebreak nor the header from your browser, as there are invisible by default)
This is a "pageheader"

This is page #2.

A pagebreak and a pageheader have been inserted below. Using the class visible, they are rendered in the browser. Here is the HTML code: <div class="pagebreak visible"></div> <div class="pageheader visible"> This is a "pageheader" </div>
This is a "pageheader"

This is page #3.
